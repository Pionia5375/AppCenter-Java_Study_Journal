추상 메서드가 하나라도 선언된 클래스는 반드시 abstract 키워드를 붙여야 한다.
추상 메서드는 메서드 바디를 가지지 않음.
추상 메서드는 자식 클래스의 메서드에서 반드시 오버라이팅하여 사용하여햐 한다.

추상 클래스의 자식 메서드는 추상 메서드를 오버라이딩하지 않으면 컴파일 에러가 발생한다.
혹은 자식 클래스 또한 abstract 키워드를 붙여 추상 클래스로 선언해야 한다.

인터페이스의 모든 메서드는 public abstract 이다.
인터페이스의 모든 변수는 public static final 이다.
모ㅅ든 속성값은 생략되어있다.

인터페이스에서는 상수로써의 변수 선언이 가능하다.
ex) int MAX_SPEED = 100; -> public static final int MAX_SPEED = 100; 

인터페이스에서는 상속을 구현이라고 표현함. -> 부모의 메서드를 구현하기 때문.

인터페이스를 구현할 때는 extends 대신 implements 키워드를 사용한다.

모든 메서드가 추상 메서드인 경우 -> 인터페이스
하나의 추상 메서드만 있는 경우 -> 추상 클래스

일반 클래스 상속과는 달리, 인터페이스는 다중 상속이 가능하다.

인터페이스를 통해 다형성을 구현할 경우, 다이아몬드 문제가 발생하지 않는다. (자식 클래스에ㅓ 모든 추상 메서드를 구현하기 때문.)
OCP원칙 open closed principle -> 개방-폐쇄의 원칙
개방에는 열려있고, 수정에는 닫혀있어야 한다.

새로운 기능을 추가할 때, 기존 코드를 수정하지 않고, 인터페이스를 구현하여 기능을 추가한다.
기존의 코드는 그대로 두고, 인터페이스를 구현하여 기능을 추가한다.

전략 패턴
클라이언트의 코드를 수정하지 않고, 인터페이스를 구현하여 기능을 추가한다.
기존의 코드는 그대로 두고, 인터페이스를 구현하여 기능을 추가한다.
