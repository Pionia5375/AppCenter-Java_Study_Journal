@Embeddable / @Embedded                                                                                                                                                                                                
                                                                                                                                                                                                                       
  엔티티가 아닌 값 타입 객체를 테이블 컬럼에 매핑할 때 씁니다. 별도 테이블 없이 부모 엔티티의 테이블에 포함됩니다.                                                                                                       
                                                                                                                                                                                                                       
  예시                                                                                                                                                                                                                   
                                                                                                                                                                                                                       
  @Embeddable 없이 하면:

  @Entity
  public class Member {
      private String city;
      private String street;
      private String zipcode;
  }

  주소 관련 필드가 여기저기 흩어지고, 다른 엔티티에서도 같은 필드를 반복해야 합니다.

  @Embeddable로 묶으면:

  @Embeddable
  public class Address {
      private String city;
      private String street;
      private String zipcode;
  }

  @Entity
  public class Member {
      @Embedded
      private Address address;  // city, street, zipcode 컬럼으로 매핑됨
  }

  DB 테이블은 똑같이 member 테이블에 city, street, zipcode 컬럼으로 들어갑니다. 테이블 구조는 변하지 않고, 자바 코드만 깔끔해지는 겁니다.

  핵심 정리
  ┌─────────────┬────────────────────────┬────────────────────────────────┐
  │ 어노테이션  │          위치          │              역할              │
  ├─────────────┼────────────────────────┼────────────────────────────────┤
  │ @Embeddable │ 값 타입 클래스에       │ "나는 내장될 수 있는 타입이야" │
  ├─────────────┼────────────────────────┼────────────────────────────────┤
  │ @Embedded   │ 사용하는 엔티티 필드에 │ "이 필드를 내장해서 매핑해"    │
  └─────────────┴────────────────────────┴────────────────────────────────┘
  언제 쓰나?

  - 주소(city, street, zipcode)처럼 의미상 하나로 묶이는 필드 그룹
  - 여러 엔티티에서 재사용하고 싶을 때
  - 별도 테이블/PK 없이 부모 테이블에 포함시키고 싶을 때
