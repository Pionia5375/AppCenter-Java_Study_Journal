테스트 코드 관련 어노테이션
- @SpringBootTest: 메인 어플리케이션 클래스에 추가하는 어노테이션인 @SpringBootApplication이 포함된 클래스를 찾고, 해당 클래스에 포함된 빈을 찾은 뒤, 테스트용 어플리케이션 컨텍스트를 만듬.
- @AutoConfigureMockMvc: MockMvc를 를 자동으로 생성 및 구성. 어플리케이션을 서버에 배포하지 않고도 테스트용 MVC 환경을 만들어줌. -> 컨트롤러 테스트용.

MVC: Model-View-Controller
Model: 데이터를 처리하는 객체
View: 데이터를 표시하는 객체
Controller: Model과 View를 연결하는 객체  

테스트 코드 관련 메서드
- perform(): 요청을 전송. 결과로 ResultActions 객체를 받으며, 개개체는 반환값을 검증하고 확인하는 andExpect() 메서드를 사용할 수 있음.
- accept(): 요청을 보낼 때 무슨 타입으로 응답을 받을지 결정. 
- andExpect(): 응답을 검증. isOk()의 경우 응답코드가 200인지 확인. 
- jsonPath("$[0].${field}"): JSON 응답의 특정 필드의 값을 가져옴. 0번째 배열에 있는 key의 field 값을 가져오고 저장된 값과 같은지 확인.

레코드: 데이터 전달을 목적으로 하는 객체를 더 빠르고 간편하게 만들기 위한 기능
-> 레코드를 사용하면, 필드, 생성자, 게터, equals(), hashCode(), toString() 메서드를 자동으로 생성해준다.

ObjectMapper: Jackson 라이브러리에서 제공하는 클래스. 객체와 json 간의 변환을 처리해준다.
예시:
Code code = new Code(13)
objectMapper.writeValueAsString(code) 
-> {'value': 13}

테스트 코드 관련 어노테이션
- @DisplayName: 테스트 메서드의 이름을 명시
- @Test: 테스트 메서드임을 명시
- @BeforeEach: 테스트 메서드가 실행되기 전에 실행되는 메서드
- @AfterEach: 테스트 메서드가 실행된 후에 실행되는 메서드
- @BeforeAll: 테스트 메서드가 실행되기 전에 실행되는 메서드
- @AfterAll: 테스트 메서드가 실행된 후에 실행되는 메서드

실행 순서

1. @DisplayName
2. @Test
3. @BeforeAll: 테스트 시작 전에 한 번만 실행. 데이터베이스 연결 및 환경 초기화에 사용
4. @BeforeEach: 테스트 메서드가 실행되기 전에 실행되는 메서드
5. @AfterEach: 테스트 메서드가 실행된 후에 실행되는 메서드
6. @AfterAll: 테스트를 모두 마치고, 공통적으로 사용하는 자원을 해제할 때 사용
